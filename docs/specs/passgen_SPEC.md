# passgen 超詳細設計仕様書（SPEC）

## 1. 概要

passgen は、日常利用に最適化されたシンプルかつ安全なパスワード生成 CLI
ツールである。

設計思想： - デフォルトで安全 - 最小限のCLI - URL-safe文字 -
曖昧文字除外 - clipboard前提 - configファイルなし（環境変数のみ）

------------------------------------------------------------------------

## 2. コマンド仕様

### コマンド

passgen \[length\]

### デフォルト

-   length: 20
-   lower/upper/digits/symbols 各1以上
-   clipboardコピーあり
-   stdout出力あり

------------------------------------------------------------------------

## 3. フラグ

--no-copy\
--no-print\
--symbols\
--digits\
--upper\
--lower\
--exclude

### 制約

--no-copy と --no-print の同時指定は禁止

------------------------------------------------------------------------

## 4. 環境変数

-   PASSGEN_LENGTH
-   PASSGEN_SYMBOLS
-   PASSGEN_DIGITS
-   PASSGEN_UPPER
-   PASSGEN_LOWER
-   PASSGEN_EXCLUDE

優先順位： CLI \> 環境変数 \> デフォルト

------------------------------------------------------------------------

## 5. 文字セット

lower: abcdefghijkmnopqrstuvwxyz\
upper: ABCDEFGHJKLMNPQRSTUVWXYZ\
digits: 23456789\
symbols: -\_.\~

※ ambiguous文字は除外

------------------------------------------------------------------------

## 6. 生成ロジック

1.  各カテゴリ最低数を生成
2.  残りを全体から生成
3.  シャッフル

乱数は crypto/rand を使用

------------------------------------------------------------------------

## 7. エラー条件

-   length \<= 0
-   必須文字数合計 \> length
-   exclude により文字集合が空

------------------------------------------------------------------------

## 8. clipboard

-   macOS: pbcopy 使用

------------------------------------------------------------------------

## 9. 補完

passgen completion zsh

.zshrc: eval "\$(passgen completion zsh --short)"

------------------------------------------------------------------------

## 10. 配布

-   goreleaser
-   Homebrew tap
-   GitHub App token 使用

------------------------------------------------------------------------

## 11. 非対応

-   configファイル
-   count
-   ambiguousフラグ
